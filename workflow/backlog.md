# Project Backlog

> Sprints are tracked here. Each sprint is a milestone-level task group.
> For complex sprints, run `/sprint-plan` to produce a detail doc in `workflow/sprints/`.
> Run `/sprint-execute` to execute a detail doc task-by-task.

---

## How This Works

- **`IN FLIGHT`** → Show remaining tasks only.
- **`READY`** → Can start immediately.
- **`BLOCKED`** → Waiting on something; note what.
- **`DONE`** → Closed.

Sprints run sequentially. Do not start Sprint N+1 until Sprint N is `DONE`.
